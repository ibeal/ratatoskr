# Unusual naming convention

- ID: `#1`
- Source: GitHub issue `https://github.com/ibeal/ratatoskr/issues/1`
- Status: in_progress
- Phase: build
- Created: 2026-07-17
- Updated: 2026-07-17

## Spec

### Goal

Align Ratatoskr's on-disk naming with the issue's requested committed-config convention.

### Acceptance criteria

1. Scope config files use `rata.toml` instead of `.rata/.rata.toml`.
2. Local scope discovery finds layered scopes by walking ancestor directories for `rata.toml`.
3. The default remote-file cache location becomes `.rata/remotes/` under the owning scope root.
4. The default global root becomes `~/.rata`, with config read from `~/.rata/rata.toml`.
5. CLI scaffolding, resolution, tests, and docs all reflect the new layout consistently.

### Non-goals

- Changing the CLI command name.
- Adding new local mutable state beyond using `.rata/` for caches.

## Intake

### Recommendation

Go.

### Reasoning

- The issue improves the committed-vs-local-state split without changing Ratatoskr's core model.
- `rata.toml` at the scope root is a better fit for committed project configuration.
- Keeping caches under `.rata/` leaves room for future local-only state and a simple `.gitignore`.
